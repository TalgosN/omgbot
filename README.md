# OMG Bot

Внутренний Telegram-бот сети клубов: смены, расписание, KPI, задачи, финансы, расходники и служебные рассылки.

## Запуск через Docker

1. Скопируйте `.env.example` в `.env` и заполните значения.
2. Положите Google service account в `key/omgbot-430116-e9a4d9c69b7f.json`.
3. Проверьте наличие рабочих файлов `data/clubs.json` и `data/phrases.json`.
4. Запустите:

```powershell
docker compose up -d --build
```

Посмотреть журнал:

```powershell
docker compose logs -f bot
```

Остановить:

```powershell
docker compose down
```

`.env`, ключ Google, SQLite, отчёты и содержимое `dump` исключены из Docker-образа. Compose передаёт настройки контейнеру через `env_file` и подключает рабочие каталоги как volumes.

## Обязательная конфигурация

- `TELEGRAM_API_KEY` — токен Telegram-бота;
- `SHIFTON_API_URL`, `SHIFTON_API_TOKEN` — API OMG Shift;
- `CHAT_REPORTS`, `CHAT_MAIN_GROUP`, `CHAT_REPAIR_EXTRA`, `CHAT_ME` — идентификаторы рабочих чатов.

При отсутствии обязательной настройки бот завершится с сообщением, в котором перечислены отсутствующие переменные. Это позволяет Docker перезапустить контейнер после исправления конфигурации вместо запуска частично работающего процесса.

## Уведомления об изменении расписания

При запуске бот передаёт OMG Shift пары `login`/`chatid` действующих сотрудников из таблицы `users_new`. Синхронизация повторяется ежедневно в 04:30 по Москве и при выполнении сотрудником `/start`.

Очередь `/api/bot/notifications/claim` проверяется каждые 15 секунд. После отправки Telegram-сообщения результат передаётся в `/api/bot/notifications/complete`. Одновременно может работать только один обработчик очереди.

## Проверки

Тесты используют только стандартный `unittest` и не обращаются к рабочим API или базе:

```powershell
python -m unittest discover -s tests -v
python -m py_compile *.py
```

Они проверяют маршрутизацию KPI, границы номеров сертификатов и абонементов, дробные часы `#двойная`, безопасную запись пользовательского текста в SQLite и подтверждение уведомлений OMG Shift.

## Данные и резервное копирование

Рабочая база находится в `db/omgbot.sql`. Перед обновлением рекомендуется остановить контейнер и скопировать весь каталог `db`:

```powershell
docker compose stop bot
Copy-Item db ("db-backup-" + (Get-Date -Format "yyyyMMdd-HHmmss")) -Recurse
docker compose start bot
```

Не копируйте активно изменяемый SQLite-файл обычной файловой командой без остановки процесса.

## Безопасность

- Не добавляйте `.env`, каталог `key` или рабочую базу в Git.
- После случайной публикации токена недостаточно удалить его из файла: токен необходимо перевыпустить у поставщика.
- Для Telegram токен перевыпускается через BotFather, после чего обновляется `TELEGRAM_API_KEY` в `.env` и перезапускается контейнер.
